# Guía de grabación general

## Checklist para preparar la sesión
- [x] Micrófono con reducción de ruido activada, nivel de entrada constante.
- [x] Cámara o ventana de pantalla grabada a 1920×1080, 30 fps.
- [x] Iluminación neutra, evitar reflejos en la pantalla.
- [x] Terminal y editor con fuente >= 16px y modo oscuro para contraste.
- [x] Cursor/puntero visible y resaltar línea cuando se ejecuten comandos clave.
- [x] Fondo limpio, evitar notificaciones o aplicaciones emergentes.

## Guion por secciones (aprox.)
1. **Introducción (00:00–00:30)** – Presentar la Lección 1 y el objetivo general: establecer un entorno MLflow listo para uso.
2. **Setup y verificación (00:30–03:30)** – Crear venv, instalar dependencias y correr `mlflow --version`.
3. **Puesta en marcha de la UI (03:30–06:30)** – Lanzar `mlflow server`, abrir `http://localhost:5000` y comentar la UI.
4. **Logging y comprobaciones (06:30–10:00)** – Ejecutar `train.py` para generar runs y abrir la vista de runs.
5. **Cierre y reflexión (10:00–11:00)** – Resumen de checkpoints y mini ejercicio de reflexión.

## Consejos de voz y ambiente
- Mantén un ritmo pausado, sin saturar de información técnica en una sola oración.
- Usa tonos ascendentes para introducir comandos nuevos.
- Pausa brevemente después de ejecutar cada comando y antes de mostrar resultados en pantalla.
